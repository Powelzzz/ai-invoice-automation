# Non-Technical Users Guide

## What This Automation Does

This automation helps process invoices faster and with less manual work.

When an invoice is uploaded, the system reads the document, pulls out the important details, records them in a tracking sheet, and sends a notification when the process is finished.

The goal is to reduce repetitive data entry and make invoice handling more consistent.

## How the User Uploads an Invoice

The user places an invoice file in the assigned Google Drive folder.

The invoice can usually be a PDF or an image file, depending on how the process is set up. The user does not need to enter the invoice details manually at this stage. Uploading the file is the main action required.

## What Happens After Upload

After the invoice is uploaded, the automation starts automatically.

It goes through a simple process:

1. The system detects the new invoice file.
2. It reads the contents of the invoice.
3. It finds key details such as the supplier name, invoice number, dates, and amount.
4. It saves those details into a shared tracking sheet.
5. It sends a message to confirm whether the invoice was processed successfully or needs review.

If the invoice is unclear or missing important information, the process can flag it for follow-up instead of recording incorrect data.

## What Output the User Receives

After the invoice is processed, the user or team receives:

- A new invoice record in Google Sheets
- A notification email showing whether the invoice was processed successfully
- A clear signal if the invoice needs manual review

This gives the team a simple way to see what has been received and what still needs attention.

## Benefits to the Finance Team

- Less manual data entry
- Faster invoice handling
- More consistent records
- Better visibility into invoice status
- Fewer missed or delayed follow-ups

Overall, this automation helps the finance team spend less time on repetitive admin work and more time on review, control, and decision-making.
